# このドキュメントについて
「効率」について考察した文書である。

# 基本用語

## 効率
効率(Efficiency)とは下記の式で表す。

```
(効率) = (成果) / (労力)
(労力) > 0
(成果) >= 0
```

英語でも記しておく。

```
(Efficiency) = (Output) / (Cost)
```

効率とは定量的尺度であるとは限らない。上記式も概念的なイメージにすぎない。この式が表しているのは、成果が大きいほど効率が高くなること、労力が小さいほど効率が高くなることなどである。

## 作業
作業とは、ある成果を上げるために必要な行動を指す。

作業には以下が存在する。

- 直接作業
- 間接作業
  - 準備作業
  - 突発作業
  - ……

直接作業とは毎回必ず必要となる行動。ただし効率化により不要となることはある。

間接作業とは直接作業でない行動。準備作業や突発作業などがある。

準備作業とは環境次第で要否が変化するが、必要な環境下では必要となる行動。たとえば PC で行う作業には「PC のある場所に行く」「PC の前に座る」といった行動が必要であり、これは準備作業にあたる。

突発作業とは普段は不要だが突発的に必要となる行動。たとえば PC を使う作業において PC、ネットワーク、利用サービスなどが故障した場合はメンテナンスや代替手段の検討が必要となるが、これは突発作業にあたる。

効率論で扱うのは直接作業のみである。

本書では特に断りがなければ、作業という言葉は直接作業を指すことにする。

### 作業の三条件
ある「成果を上げるために必要な行動」が作業であるためには、以下を満たす必要がある。

- 過程化できる(Processable)
  - 過程(プロセス/Process)として第三者が理解できる言葉で記述できる
- 繰り返せる(Repeatable)
  - 過程を複数回繰り返すことができる
- 有限である(Finite)
  - 「成果が出る」というゴールがある

## 労力
労力(Cost)とは直接作業に絡む労力を指す。

たとえば「メールのインボックスからスパムメールを削除する作業」を考えた場合、労力とはインボックス中のメールを読んでスパムかどうかの判断、スパムだったメールを削除する操作、次のメールを読みに行く操作などにかかるものを指す。逆に PC の起動、メーラー変更に伴う設定変更、ルーターの故障に伴う交換作業、スパムメール中の添付ファイルをうっかり開いたことによるウイルス被害に伴う復旧作業などは労力とは言わない。

### ゼロコスト
ゼロコスト、つまりは `労力 = 0` は存在しない。あらゆる作業には少なからず労力がする。

仮にゼロコストで成果を上げることができた場合、それはもはや効率化とは呼ばない。効率論の範疇外とする。

ちなみにゼロコストで成果を上げること自体は可能である。自動化や委譲といった方法がある。ただし、これら方法についても間接作業に関する労力は発生する(あるいは発生する可能性がある)。

## 効率化
効率化(Efficy)とは効率を高めること。

Efficy は造語である。2019/03 現在、英語圏に効率化という概念を表す英単語は存在しない。一応 Streamline はあるものの意味が直感的ではない。また Optimize や Improve といった別用語も存在するものの、これらは広義である。「効率化」という概念を表す専用の用語に対する適切な英語は、存在しない。ゆえに Efficy という造語をつくるに至った。

## 効率論
効率論(Effisology)とは効率化について扱う分野。

## 効率主義と効率家
効率主義(Efficism)とは効率化に傾注する主義。

効率化(Efficist)とは効率主義を持つ者。
